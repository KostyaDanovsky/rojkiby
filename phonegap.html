Why PhoneGap?

Your buisness is doing good. You decide to go mobile and help your clients To easily access your services. Well, that's a good decision, but there's one problem. Some of your clients use iOS, some Android, some valuable users are big fans of Microsoft technologies. So there's no option for you and you need to implement application for all these platforms. Well, cross-platform mobile development can become a tiring time- and money-consuming routine. But what should you do if you need to kickstart your application quickly and with minimal risks? The answer is PhoneGap.

What is PhoneGap?

As mentioned on official PhoneGap website, PhoneGap is an open source framework for quickly building cross-platform mobile apps using HTML5, Javascript and CSS. Using familiar technologies and PhoneGap an average web-developer can create amazing applications that will run on different platforms smoothly. If your application requires access to camera, speaker, gps etc. that's also possible through plugins. There are thousands of plugins available on the web that will cover most of your needs. As well you write plugins yourself if need to use some native functions in you app. There are also several good PhoneGap frameworks like Ionic that will let you use some native-like JS controls with appropriate CSS styling. Moreover, today it's also possible to mix native and html controls using frameworks like Supersonic. In other words, community is big enough and it's expanding fast.

What are pros?

PhoneGap is an ideal choice to make your application if you need it done as fast as possible and you don’t want to waste your money (in other words if you need to quickly create a prototype).
It's very unlikely, that one developer (team) will be making both your iOS and Android application. That's why you will need to double the investments for domain area investigation. Moreover most of the features should be duplicated on iOS and Android platforms. To develop your app with PhoneGap you need one team, one tool to run the application on multiple platforms.
Using PhoneGap you can reach a wider audience - it is really easy to launch your app on any of the supported platforms. Once written, the code can become an android, ios, blackberry and windows phone application or… even a mobile version of your website.
PhoneGap application is a perfect combination of modern front-end technologies and wonderful features your phone or tablet provides.

What are cons?

Everything has it’s drawbacks and, unfortunately, PhoneGap isn’t the exception. Today hybrid mobile applications (such as PhoneGap ones) still can’t be as fast as native ones. But they will be! Consider desktop: most of native applications are replaced by web analogs. Some people even use Google docs instead of MS Office! So for mobiles it's just a question of time.
It can’t be said that PhoneGap is a suitable tool for creating the application that relies heavily on graphics or needs to be lightning-fast. So in most cases it would be a bad idea to develop game using this technology. And though PhoneGap makes cross-platform development as easy as it can be, platform-specific issues will still take your time.

So, why PhoneGap?

You have already read about PhoneGap’s pros and cons and at this moment understand that it is a perfect tool for creating really beautiful mobile applications really fast. Yes, PhoneGap isn’t ideal but as any tool in reliable hands it can become as close to ideal as possible.
